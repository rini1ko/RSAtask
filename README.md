# RSA

Захищений термінальний чат, який працює за принципом клієнт-сервер. Для безпеки спілкування реалізовано такі кроки:

Щоб не передавати пароль від чату у відкритому вигляді, ми використовуємо RSA. Клієнт генерує пару ключів і відправляє публічний ключ серверу. Сервер вигадує спільний секретний пароль для чату, шифрує його публічним ключем клієнта і віддає назад.

Оскільки RSA працює повільно, самі повідомлення в чаті шифруються тим самим спільним секретним паролем. Для цього написана функція xor_cipher.

Щоб переконатися, що повідомлення не змінилося по дорозі, перед шифруванням для нього рахується SHA-256 хеш (використано стандартний модуль hashlib). Отримувач порівнює хеш присланого тексту зі своїм.

Чат працює через TCP-сокети. Щоб користувач міг одночасно читати чужі повідомлення і набирати свої, клієнтська частина розділена на кілька потоків за допомогою модуля threading. Сервер просто пересилає зашифровані дані від одного клієнта до інших.

Так то в сервері чату пусто, але можна розкоментувати лінію коду щоб побачити, що повідомлення зашифровані при передачі(# print(f"[msg sent]: {msg.decode()}" в handle_client).

Для запуску програми потрібно в одному терміналі запустити server.py, і в двох інших client.py. В клієнта питатимуть ім'я, яке потім буде показуватись в чаті.
